================================================================================
BEHAVIOR 2.2.2: INDEX INTEGRITY VERIFICATION (索引完整性验证)
================================================================================

GOAL: 验证索引和关联关系的完整性

RECOMMENDED AGENT: Define-Agent
  - 该 behavior 涉及数据关系检查、引用完整性验证
  - Define-Agent 擅长执行 Python 代码进行数据关系分析

================================================================================
ACTIONS:
================================================================================

ACTION 2.2.2.1: Verify Primary and Foreign Keys
  验证主键和外键的完整性
  - 检查主键的唯一性和非空性
  - 验证外键的有效性
  - 识别键约束违反情况

ACTION 2.2.2.2: Check Referential Integrity
  检查关联表之间的引用完整性
  - 验证表间引用的有效性
  - 检查级联关系的完整性
  - 识别引用完整性违反

ACTION 2.2.2.3: Identify Orphaned Records
  识别孤立记录和断链问题
  - 发现无效的外键引用
  - 识别孤立的子记录
  - 检测断裂的关联链

================================================================================
OUTPUT ARTIFACTS:
================================================================================
- index_integrity_analysis: 索引完整性分析
- referential_integrity_check: 引用完整性检查结果
- orphaned_records_list: 孤立记录清单
