================================================================================
BEHAVIOR DE.1.3: VALIDATION AND ACCEPTANCE
================================================================================

GOAL: Validate all data collection and inventory activities meet acceptance criteria and establish data existence foundation.

RECOMMENDED AGENT: PCS-Agent

================================================================================
ACTIONS:
================================================================================

ACTION DE.1.3.1: Artifact Completeness Check
  Verify all required artifacts are generated and complete
  - Confirm data_source_register exists and is non-empty
  - Confirm data_collection_log documents all acquisitions
  - Confirm data_inventory_report contains required statistics
  - Confirm data_catalog_metadata is comprehensive

ACTION DE.1.3.2: Data Quality Gate Check
  Validate data meets minimum quality requirements
  - Verify data volume sufficient for analysis
  - Check data completeness above minimum threshold
  - Validate data accessibility and format correctness
  - Confirm no critical data issues blocking progress

ACTION DE.1.3.3: PCS Framework Alignment
  Verify alignment with PCS framework requirements
  - Predictability: Confirm data sufficiency for prediction
  - Computability: Verify data processing reproducibility
  - Stability: Validate data consistency and traceability

================================================================================
OUTPUT ARTIFACTS:
================================================================================
- step_validation_report (pass/fail status for all acceptance criteria)

  
================================================================================
TASK TEMPLATE:
================================================================================
- Mark each criterion as PASS/FAIL with evidence
- Document any issues or gaps requiring attention
- Provide recommendation: PROCEED to next step or REWORK current step
- Persist validation report under ./outputs/stage_1_data_existence/step_1/

================================================================================
ACCEPTANCE EXAMPLES:
================================================================================
- step_validation_report exists with status for each acceptance criterion
- All critical acceptance criteria marked as PASS
- Any FAIL items have documented remediation plans
- PCS framework alignment confirmed for this step
- Recommendation provided with clear rationale
